Project lifecycle — one Claude project, multiple roles, full idea lifespan
A Claude project is created once per idea and shared across roles. Each role loads the same L0 + L2 context and the relevant L1 skill for their phase. No context is lost between phases — the project accumulates shared understanding over time.
Project setup — what goes where when creating a new idea project
Project Instructions (always)
  • IDEA-XXX + title
  • JPD URL + Epic URL
  • Active skills listed explicitly
  • Idea context if short and stable
MD file (complex ideas only)
  • Scope boundary
  • Key decisions locked
  • Open questions
  • Key contacts / dependencies
Never in project storage
  • Current Jira state
  • Figma component details
  • Any live data snapshot
  • Team roster (lives in L0)
Example: Project Instructions template
Copy this template when creating a new idea project. Replace the placeholders. Skills listed here tell Claude what to load and when.
# Project: [IDEA-XXX] — [Idea Title] ## Identity JPD idea: [JPD URL] Epic: [Jira Epic URL — add once created] Product: [e.g. Energy System Designer] ## Always active # Load these for every task in this project. - solargis-company-context # company, all teams, cross-team ownership, product index - solargis-esd-context # deep ESD product context — replace with your product's L2 skill ## Phase skills — load when the task matches # Claude selects the relevant skill based on what is being worked on. - jpd-idea-description # Phase 1 — writing or refining the JPD idea - epic-wbs-breakdown # Phase 2 — breaking the epic into Jira issues - jira-story-writing # Phase 3 — writing stories, tasks, analyses - release-go2market # Phase 4 — go-to-market doc in Confluence - technical-writer-prep # Phase 5 — TW briefing page in Confluence - release-linkedin-post # Phase 6 — LinkedIn release announcement ## Idea context # Short idea: write scope and decisions here directly. # Complex idea: attach idea-context.md to this project instead. Scope: [1–2 sentences] Out of scope: [explicit exclusions] Key decisions locked: [decisions that should not be reopened] Open questions: [unresolved items — expected to evolve] ## Live data # Always fetch current state via MCP — never assume from memory. - Jira: query live for story status, sprint, blockers - Figma: fetch component when referenced - Solargis KB: use web_fetch on docs URL when official documentation is needed
